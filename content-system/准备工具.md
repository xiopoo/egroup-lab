## 四、品牌指南

### 4.1 文案风格

| 项目 | 规则 |
|------|------|
| 语言 | 中文，口语化，避免专业术语堆砌 |
| 语气 | 真诚、专业、有同理心，不居高临下 |
| 人称 | 多用"你"，少用"我们"，建立对话感 |
| 结构 | 首段悬念式开篇，末段三点式总结，短段落（2-4句），清晰小标题 |
| 禁忌 | 不使用绝对化承诺（100%、一定），不贬低同行 |
| 免责 | 每篇文章结尾必须包含免责声明 |

### 4.2 视觉风格

| 项目 | 规则 |
|------|------|
| 品牌色 | 仅h1、h2使用#AB1942，h3使用#2D2D2D加粗 |
| 字体 | 正文15px，行高1.9，首行缩进2em |
| 分隔线 | 60px宽，3px高，#333333居中 |
| 标题线 | h2底部2px渐变线（透明→#AB1942→透明） |
| 引用 | 左侧2px #AB1942竖线，白色背景 |
| 品牌推广区 | 浅红背景#FDF6F7，圆角12px，居中 |
| 页面背景 | #FAFAF8 |

### 4.3 发布规范

| 项目 | 规则 |
|------|------|
| 频率 | 每周1-2篇，固定时间发布 |
| 标题 | 痛点+解决方案，15-20字 |
| 封面 | 品牌色调，按需生成 |
| 标签 | 每篇3-5个关键词，入库时同步概念关联 |
| 审核流程 | AI检测 → 法律审查 → 首尾段优化 → 排版 → 发布 → 入库拆解概念 |

## 六、自助诊断系统（已上线）

### 6.1 问卷结构

| 模块 | 问题数量 | 类型 |
|------|----------|------|

| 法律风险 | 3题 | 单选/多选 |
| 家庭资产 | 3题 | 单选/多选 |

### 6.2 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 债务压力 | 30% | 负债率、到期时间、综合成本 |
| 资产质量 | 20% | 变现能力、抵押情况、贬值程度 |
| 经营健康 | 25% | 盈利能力、现金流、核心业务 |
| 法律风险 | 15% | 涉诉情况、担保责任、刑事风险 |
| 家庭保障 | 10% | 资产隔离、共债情况 |

### 6.3 输出报告

| 项目 | 内容 |
|------|------|

| 风险雷达图 | 五个维度可视化 |
| 关键发现 | 3-5条核心问题 |
| 建议路径 | 对应解决方案（庭外和解/债务重组/司法重整等） |
| 下一步行动 | 联系我们的方式 |

## 八、文件管理

### 8.1 目录结构

```
hj/
├── .trae/skills/            # Trae 技能体系（14个 Skill，项目唯一真源）
│   ├── hj-publish/          # 全流程发布引擎（含入库）
│   ├── hj-content/          # 内容五维诊断
│   ├── hj-cases/            # 案例库检索
│   ├── hj-wechat-html/      # 公众号HTML排版（品牌烙印模板）
│   └── ...
├── website/                 # 官网模块
│   └── public/              # 唯一正式网站源代码
└── content-system/          # 公众号内容模块
    ├── content/             # 内容库
    │   ├── articles/        # 文章摘要（含章节结构、核心观点）
    │   ├── concepts/        # 核心概念
    │   ├── quotes/          # 金句库
    │   └── index.json       # 完整索引
    ├── output/              # 输出文件
    │   ├── 公众号HTML输出/   # 公众号版HTML
    │   └── *.md             # Markdown源文件
    └── 准备工具.md           # 本文件（持续更新）
```

### 8.2 更新规则

- 每发布一篇文章后入库，更新内容库和索引
- 网站迭代后更新本工具的状态
- SEO等持续优化按月追踪

# 灰金困境自助诊断系统 - 微信公众号集成方案

## 一、核心目标

将现有的企业困境自助诊断系统集成到微信公众号，创建专属栏目供客户直接使用，实现"公众号入口 → H5诊断 → 报告推送 → 咨询对接"的完整流程。

## 二、技术架构设计（轻量级）

### 2.1 整体架构图

```
微信公众号
    │
    ├─ 自定义菜单：【自助诊断】
    │
    ├─ OAuth授权（用户身份识别）
    │
    └─ 跳转至H5诊断页面
            │
            ├─ 前端：单页面应用（HTML+CSS+JS）
            │       ├─ 诊断问卷组件
            │       ├─ 进度条组件
            │       ├─ 结果展示组件（雷达图）
            │       └─ 咨询预约组件
            │
            └─ 后端：Serverless函数（Node.js）
                    ├─ 评分算法API
                    ├─ 结果存储API
                    ├─ 微信消息推送API
                    └─ 用户信息管理API
```

### 2.2 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | HTML5 + TailwindCSS + Vue3（CDN引入） | 零构建、轻量级、微信内兼容性好 |
| 后端 | Vercel Serverless Functions | 无需服务器、自动扩缩容、免费额度足够 |
| 数据库 | Supabase（PostgreSQL） | 免费额度足够、自动备份、RESTful API |
| 微信对接 | 微信公众号API（OAuth2.0） | 标准对接方式 |
| 图表 | Chart.js（CDN引入） | 轻量级、移动端适配好 |

### 2.3 关键模块

**模块1：微信OAuth授权**
- 用户点击菜单 → 跳转授权页 → 获取openid → 进入诊断页面

**模块2：诊断问卷引擎**
- 层次推进结构（6个层次，共36题）
- 实时进度显示
- 本地存储中途退出支持

**模块3：评分算法**
- 五维度加权评分（债务30%、经营25%、资产20%、法律15%、家庭10%）
- 等级判定（健康/关注/预警/危机）
- 报告自动生成

**模块4：结果存储与推送**
- 诊断结果存入数据库
- 通过客服消息推送报告链接
- 支持报告查询和分享

**模块5：咨询对接**
- 诊断完成后显示咨询预约表单
- 表单提交后发送通知给运营人员

## 四、用户身份验证机制

### 4.1 认证流程

```
用户进入公众号
    │
    ├─ 点击【自助诊断】菜单
    │
    ├─ 跳转微信OAuth授权页
    │       ├─ 首次访问：弹出授权窗口
    │       └─ 再次访问：自动授权（静默模式）
    │
    ├─ 获取用户openid（唯一标识）
    │
    └─ 进入诊断页面（已登录状态）
```

### 4.2 用户数据存储

| 字段 | 类型 | 说明 |
|------|------|------|
| openid | String | 用户唯一标识（微信提供） |
| nickname | String | 用户昵称 |
| avatar | String | 用户头像URL |
| created_at | DateTime | 首次访问时间 |
| last_diagnosis_at | DateTime | 最近诊断时间 |
| diagnosis_count | Integer | 诊断次数 |

### 4.3 安全措施

- 仅存储必要的用户信息（openid、昵称、头像）
- 不存储用户诊断的敏感财务数据（仅存储评分结果）
- 所有数据传输使用HTTPS
- 定期清理超过90天的过期诊断记录

## 五、诊断数据的安全传输方案

### 5.1 数据传输流程

```
用户填写答案（前端）
    │
    ├─ 加密：AES-256-CBC加密
    │
    ├─ 传输：HTTPS POST请求
    │
    ├─ 后端解密：AES-256-CBC解密
    │
    ├─ 计算评分：服务器端计算
    │
    ├─ 存储结果：加密存储到数据库
    │
    └─ 返回报告：加密传输回前端
```

### 5.2 安全措施

| 措施 | 说明 |
|------|------|
| HTTPS传输 | 所有接口强制HTTPS，防止数据泄露 |
| AES加密 | 请求体使用AES-256-CBC加密，密钥定期轮换 |
| 输入校验 | 后端对所有输入进行严格校验，防止注入攻击 |
| 限流保护 | 单用户每分钟最多提交3次请求 |
| 敏感数据隔离 | 用户财务数据不存储，仅存储评分结果 |
| 定期备份 | 数据库每日自动备份，保留7天 |

## 八、关键技术难点

### 8.1 OAuth授权流程
- **难点**：微信授权回调URL必须与公众号配置一致，且必须是HTTPS
- **解决方案**：使用Vercel提供的HTTPS域名，在公众号后台准确配置授权域名

### 8.2 微信内H5兼容性
- **难点**：微信内置浏览器对某些JS API支持不完善
- **解决方案**：使用纯前端技术栈（Vue3 CDN + TailwindCSS），避免使用微信JSSDK（除非必要）

### 8.3 数据安全
- **难点**：用户财务数据敏感，需确保传输和存储安全
- **解决方案**：HTTPS传输 + AES加密 + 敏感数据不存储（仅存储评分结果）

### 8.4 服务器资源限制
- **难点**：Vercel免费额度有限（每月100GB流量、1000次Serverless调用）
- **解决方案**：优化前端资源（压缩图片、缓存静态文件），预计日均调用量远低于免费额度

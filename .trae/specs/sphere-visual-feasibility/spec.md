# 球形视觉元素可行性评估 Spec

## Why

当前企业核心动态关系图采用 2D 平面轨道环绕展示，6 个节点以独立速度绕圆心公转。为提升视觉冲击力与品牌辨识度，评估将平面环绕改造为 3D 球形视觉元素的可行性——文字以两字一行的格式排列在球体表面，整体构成完整的球形视觉效果。

## What Changes

本 spec 为**可行性评估报告**，不涉及代码变更。分析以下维度：
- 球形设计方案的视觉呈现可行性
- 信息传达效果与可读性评估
- 技术实现方案对比（CSS 3D / Canvas / Three.js）
- 与现有设计体系的兼容性
- 具体的球体布局方案与交互设计建议
- 潜在优化方向与实施路线图

## Impact

- Affected specs: `independent-orbit-system`（当前在用的轨道方案可能被替代或改造）
- Affected code: `public/index.html`（如确定实施，将涉及大幅改造）
- Design system: Hero 区域的整体视觉风格

## 可行性分析

### 一、核心概念定义

**球体文字布局**：将文字内容以「两字一行」的短文本块为单位，分布在 3D 球体表面的不同经纬度位置。球体整体或分层自转，使不同文字块依次经过正面/背面，形成立体环绕视觉效果。

**可用文字资产**：

| 层级 | 内容 | 是否 2 字 | 块数 |
|------|------|:---------:|:----:|
| 核心 | 企业 | ✓ | 1 |
| 内层 | 业务、财务、债务 | ✓ | 3 |
| 外层 | 资产重组、债务重组、业务重组 | ✗（4字） | 3 |
| 可扩展 | 诊断、重组、价值、重塑、破局... | ✓ | 多 |

**2 字块的拆分/重构方案**：
- 外层 4 字词 → 可拆为 2 个 2 字块：「资产」「重组」「债务」「重组」「业务」「重组」——但会产生重复的「重组」
- 或引入更多相关 2 字概念：「资产」「债务」「业务」「诊断」「重组」「破局」「价值」「重塑」

### 二、视觉呈现可行性

**优势分析**：

| 维度 | 评估 |
|------|------|
| **视觉冲击力** | 3D 球体的空间感远强于 2D 平面环绕，在 hero 区域形成强烈的视觉焦点 |
| **品牌差异化** | 中文 2 字球体布局尚属少见，可形成独特的品牌识别记忆点 |
| **层次表达** | 球体天然支持多层——核心在球心、内层在赤道附近、外层在较高纬度 |
| **动态美感** | 球体自转时文字块依次经过正面/侧面/背面，产生自然的深度运动 |

**风险与限制**：

| 风险项 | 说明 |
|--------|------|
| **文字密度不足** | 仅 6 个 2 字块在球体上显得稀疏，球体轮廓感弱 |
| **可读性挑战** | 背面文字需反转或隐藏；侧面文字倾斜后辨识度下降 |
| **中文 2 字特性** | 中文方块字比西文更适合球体排列（字形方正），但 2 字块横向仍占空间 |
| **信息可见性** | 同一时间仅 2-3 个文字块位于正面，其余被遮挡或模糊 |

**结论**：**视觉上可行但需增加文字块数量**（推荐 12-18 个 2 字块）才能形成饱满的球体轮廓。仅靠现有 6 个节点的文字不足以支撑完整的球体视觉效果。

### 三、信息传达可行性

| 场景 | 当前 2D 轨道 | 球体方案 |
|------|-------------|---------|
| **同时可见** | 全部 6 节点同时可见 | 正面仅 3-4 个字块清晰可见 |
| **层级关系** | 内外圈半径清晰区分层级 | 球心/赤道/高纬度区分层级 |
| **文字可读** | 始终正立，100% 可读 | 侧面的字块倾斜，辨识度下降 |
| **信息完整度** | 一次性获取全部信息 | 需等待旋转才能看到全部文字 |
| **交互认知** | 节点即点击目标 | 文字块即点击目标，认知一致 |

**优化策略**：
- 球体自转速度不宜过快（建议 30-45s 一圈），确保每个文字块在正面停留足够时间
- 正面的文字块放大突出（scale 1.2~1.5），背面的缩小淡出（opacity 0.2~0.3）
- 鼠标悬停时球体暂停旋转，便于用户仔细查看

**结论**：信息传达效率**低于 2D 方案**（同时展示的信息量少），但**沉浸感和品牌感更强**。适合品牌展示场景，不适合需要同时展示大量信息的场景。

### 四、技术实现方案对比

#### 方案 A：CSS 3D Transform（推荐优先评估）

**原理**：利用 CSS `perspective` + `transform-style: preserve-3d` + `rotateY() translateZ()` 将文字块定位在 3D 球体表面。

```
球体表面定位公式：
  经度 θ, 纬度 φ, 半径 R
  x = R · cos(φ) · sin(θ)
  y = R · sin(φ)
  z = R · cos(φ) · cos(θ)
通过 rotateY(θ) translateZ(R·cos(φ)) translateY(R·sin(φ)) 组合实现
```

**实现要点**：

```css
.sphere-wrap {
  perspective: 1000px;
  transform-style: preserve-3d;
}
.sphere-text-block {
  position: absolute;
  transform-style: preserve-3d;
  /* 示例：经度 0°, 纬度 0° */
  transform: rotateY(0deg) translateZ(150px);
  /* 反旋转保持文字朝向用户 */
  backface-visibility: hidden;
}
```

**性能**：GPU 加速，60fps 流畅
**兼容性**：现代浏览器支持，iOS Safari 需注意
**交互**：`pointer-events` 在 3D 空间中的命中区域与视觉一致

**优点**：
- 零依赖，纯 CSS 实现
- 性能最优（GPU 加速）
- 与现有 DOM 节点无缝配合（tooltip、点击事件）

**缺点**：
- Z-order 无法动态排序——背面文字可能叠在正面之上
- 文字反旋转逻辑复杂（每个文字需独立 counter-rotate）
- 纬度变化时 translateY 的叠加顺序敏感
- 超过 10 个 3D 元素时 Safari 渲染可能异常

**可行性评分**：★★★★☆

---

#### 方案 B：Canvas 2D 投影渲染

**原理**：在 Canvas 上手动计算每个 3D 球体文字在 2D 平面上的投影位置、大小和透明度。

```
投影计算：
  scale = focalLength / (focalLength + z)
  screenX = x * scale + centerX
  screenY = y * scale + centerY
  opacity = 0.2 + 0.8 · (z + R) / (2R)  // 根据 z 值动态变化
  fontSize = baseFontSize · scale
```

**实现架构**：

```javascript
class SphereEngine {
  constructor(canvas, options) { /* R, focalLength, textItems */ }
  update(deltaTime) { /* 球体旋转, 更新每个文字的 3D 坐标 */ }
  render() {
    // 1. 按 z 值排序（背面先绘制）
    // 2. 绘制连接线（球体经线/纬线网格）
    // 3. 绘制每个文字块（位置/大小/透明度）
    // 4. 绘制核心"企业"（始终在中心）
  }
  onHover(callback) { /* 鼠标悬停检测—需要 canvas 坐标计算 */ }
}
```

**优点**：
- 完全控制渲染顺序（z-sort 自然）
- 可实现任意视觉效果（发光、渐变、轨迹尾迹）
- 文字块数量可扩展至 30+ 不影响性能

**缺点**：
- 与现有 DOM 交互桥接——tooltip、点击面板需额外的 hit-testing 逻辑
- 文字渲染依赖 `ctx.fillText`，抗锯齿效果不如 DOM 文字
- Canvas 的 HiDPI 适配需要额外处理（devicePixelRatio）
- 开发成本最高

**可行性评分**：★★★☆☆

#### 方案 C：CSS 2.5D 椭圆投影（推荐折中方案）

**原理**：不追求真实的 3D 球体，而是在 2D 平面上通过**椭圆轨迹 + 动态 scale/opacity** 模拟球体效果。

文字块沿不同椭圆轨道运行（对应球体不同纬度），经过正面时放大、明亮，转至侧面时缩小、半透明。

```
纬度 0°（赤道）: 椭圆 a=R, b=R     → 最大轨道
纬度 30°:       椭圆 a=R, b=R·cos30° → 略扁椭圆
纬度 60°:       椭圆 a=R, b=R·cos60° → 很扁的椭圆

每个文字块沿自己的椭圆轨道独立公转
经过"正面"（轨道最低点）时 scale=1.2, opacity=1
转到"侧面"时 scale=0.8, opacity=0.4
转到"背面"（轨道最高点）时 scale=0.5, opacity=0.15
```

**关键 CSS**：

```css
/* 纬度 0° 赤道层 - 完全的圆形轨道 */
@keyframes orbit-3d-equator {
  0% { transform: rotate(0deg) translateX(50%) rotate(0deg); opacity: 0.3; }
  90deg { opacity: 1; }  /* 正面时最亮 */
  180deg { opacity: 0.3; } /* 背面 */
  270deg { opacity: 1; }
  100% { transform: rotate(360deg) translateX(50%) rotate(-360deg); opacity: 0.3; }
}

/* 纬度 45° 层 - 扁椭圆轨道（模拟透视收缩） */
@keyframes orbit-3d-lat45 {
  0% { transform: rotate(0deg) translateX(42%) scaleY(0.7) rotate(0deg); }
  100% { transform: rotate(360deg) translateX(42%) scaleY(0.7) rotate(-360deg); }
}
```

**优点**：
- 纯 CSS 实现，与现有架构完全兼容
- 无需 Canvas 渲染或 JS 引擎
- 文字为真实 DOM 节点，tooltip 和点击事件自然继承
- 可通过多层椭圆叠加出球体感

**缺点**：
- 需要多个独立轨道层（建议 3 层 × 每层 4-6 个字块）
- 椭圆 `scaleY()` 导致文字变形，需使用嵌套容器反缩放
- 背面文字的遮挡效果只能通过 opacity 模拟，无法实现真实遮挡

**可行性评分**：★★★★★

### 五、具体设计建议

#### 5.1 推荐方案：CSS 2.5D 椭圆球体

**布局设计**：

```
第 1 层（赤道，半径 55%）：诊断 · 重组 · 破局 · 重塑  ← 6 字块
第 2 层（纬度 30°，半径 45%）：业务 · 财务 · 债务 · 资产   ← 6 字块  
第 3 层（纬度 60°，半径 30%）：价值 · 秩序 · 平衡 · 增长   ← 4-6 字块
球心：企业（放大突出）
```

**视觉参数**：

| 参数 | 赤道层 | 30° 层 | 60° 层 |
|------|:------:|:------:|:------:|
| 轨道椭圆比例 | 1:1（正圆） | 1:0.85 | 1:0.5 |
| 正面字体大小 | 16px | 14px | 12px |
| 正面不透明度 | 1.0 | 0.9 | 0.7 |
| 背面不透明度 | 0.15 | 0.1 | 0.08 |
| 正面阴影 | 强 | 中 | 弱 |
| 公转速度 | 35s CW | 30s CCW | 25s CW |
| 文字块间距 | 60° | 60° | 72° |

#### 5.2 视觉增强（可选）

- **经纬网格线**：在球体外围添加半透明的经纬线 SVG（增强球体轮廓感）
- **核心光晕**：「企业」周围添加环形光晕 `radial-gradient`
- **节点颗粒**：在文字块路径上添加微小光点 trail（增强运动轨迹感）
- **进出动效**：页面滚动进入视口时，文字块从聚合→展开为球体

#### 5.3 交互设计

- **悬停**：该文字块暂停公转 + scale(1.3) + 显示 tooltip 详情
- **点击**：展开详情面板（沿用现有 diagram-panel 逻辑）
- **球体全局控制**：点击核心「企业」切换全部公转暂停/恢复
- **鼠标跟随**（增强版）：鼠标在 hero 区域内移动时，球体视角微偏转 parallax

### 六、潜在优化方向

1. **文字块扩充至 14-18 个**：引入更多企业相关 2 字概念（诊断、破局、价值、秩序、平衡、增长、重构、赋能、协同、创新），使球体更饱满
2. **动态内容加载**：根据用户交互，球体上的文字块可动态替换（如 hover 某区域显示该领域的子概念）
3. **滚动驱动动画**：球体自转速度与页面滚动位置关联，增强页面叙事感
4. **Canvas 渐进增强**：在 CSS 2.5D 版本基础上，为高性能设备提供 Canvas 3D 降级/升级选项
5. **品牌联动**：球体旋转的停顿点设计在品牌关键词（如「重组」「破局」）面向用户的时刻
6. **技术预研**：iOS Safari 的 3D 渲染限制、低端设备的降级策略

### 七、实施路线图

```
Phase 1（1-2 天）：CSS 2.5D MVP
  └─ 3 层椭圆轨道 × 4-6 字块
  └─ 正面/背面动态 scale + opacity
  └─ 保留现有 tooltip 和点击交互

Phase 2（2-3 天）：视觉增强
  └─ 经纬网格线 SVG
  └─ 核心光晕 + 节点 trail
  └─ 进出动效
  └─ 鼠标跟随 parallax

Phase 3（备选）：Canvas 进阶
  └─ 评估是否需要 Canvas 3D 渲染
  └─ 评估性能与交互桥接方案
```

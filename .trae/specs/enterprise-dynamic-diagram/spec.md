# 企业核心动态关系图 Spec

## Why

首页 hero 右侧视觉区域当前展示"困境重组与诊断"星系图（天体系统隐喻），但缺少可直接点出"企业"核心并与业务/财务/债务三维度建立直观关联的交互式图表。需要设计一个以企业为中心的动态关系图，集成诊断与重组双层结构，并加入悬停/点击交互，提升信息传达效率和用户参与度。

## Design Concept: "企业核心动态关系图"

采用同心圆布局，以"企业"为核心，内层均匀分布三大核心要素（业务·财务·债务），外层均匀分布三大重组方案（资产重组·债务重组·业务重组），通过关联线条和动效展示从诊断到重组的完整路径。

### 结构层次

```
           [业务]
             |
    [债务] —-[ 企业 ]— [财务]
             |
           [运营]
 ===== 诊断维度（内层） =====

       [资产重组]
           |
[业务重整]—[ 企业 ]—[债务重组]
           |
       [业务重组]
 ===== 重组方案（外层） =====
```

### 同心圆布局参数

| 层级 | 容器宽度 | 节点数 | 节点样式 | 节点尺寸 |
|------|---------|--------|---------|---------|
| 核心"企业" | — | 1 | 圆形 + 品牌色背景 | 90×90px |
| 第一层（诊断） | 48% | 3 | 圆角矩形 | 80×36px |
| 第二层（重组） | 76% | 3 | 圆角矩形 | 96×40px |

### 颜色方案（红色渐变背景 #AB1942→#8B1435 上）

| 元素 | 颜色 | 说明 |
|------|------|------|
| 核心"企业" | 白色背景 rgba(255,255,255,.15) + 白边框 | 主体地位 |
| 内层节点 | 各自独立渐变色 | 诊断维度区分 |
| 外层节点 | 白色半透明 rgba(255,255,255,.12) + blur | 重组方案 |
| 关联线条 | rgba(255,255,255,.2) 半透明 | 层级连接 |
| 内轨道环 | rgba(255,255,255,.15) 虚线 | 诊断范围 |
| 外轨道环 | rgba(255,255,255,.1) 虚线 | 重组范围 |

### 节点颜色明细

**第一层（诊断维度）：**
- 业务：`linear-gradient(135deg, #5B9BD5, #4A90D9)` 青蓝色
- 财务：`linear-gradient(135deg, #4EDB8A, #2ECC71)` 绿色
- 债务：`linear-gradient(135deg, #F5B041, #F39C12)` 琥珀色

**第二层（重组方案）：**
- 资产重组：白色半透明 + border
- 债务重组：白色半透明 + border
- 业务重组：白色半透明 + border

### 动效设计

1. **加载动画**：页面加载时节点从中心向外渐入展开，持续 0.6s
2. **轨道环旋转**：内轨 30s 顺时针，外轨 40s 逆时针
3. **关联线条流动**：从中心到内外节点的连接线呈流动虚线效果
4. **节点悬浮**：节点有轻微的上下浮动 (translateY ±6px, 3s 周期)

### 交互设计

| 交互 | 触发 | 效果 |
|------|------|------|
| 悬停 (hover) | 鼠标悬停任一节点 | 显示简要说明 tooltip，节点放大 1.1x |
| 点击展开 | 点击任一节点 | 在图表下方或侧边展开详细信息面板 |
| 关闭详情 | 点击关闭按钮或再次点击节点 | 信息面板收起 |

### 响应式设计

| 断点 | 行为 |
|------|------|
| ≥1024px | 完整显示双圈结构 + 交互 + tooltip |
| 768-1023px | 显示完整结构，tooltip 调整为轻量提示 |
| ≤767px | 隐藏图表（移动端不展示复杂视觉） |

## What Changes

- **方案 A**：替换首页 hero 右侧视觉区域（`public/index.html` 的 `.hero-visual`）
- CSS：新增完整关系图样式（包括动画、交互、响应式）
- HTML：替换 `.hero-visual` 内部结构
- JS（可选）：点击展开详情面板的交互逻辑

## Impact

- Affected code: `public/index.html` CSS 区块 + HTML 区块
- 仅替换 hero 右侧视觉，不影响左侧 hero-content 和其他页面
- 移动端隐藏，无性能影响

## Changed Requirements

### Requirement: 同心圆双圈布局
系统 SHALL 在首页 hero 右侧显示以"企业"为核心的同心圆双圈结构。

#### Scenario: 桌面端显示
- **WHEN** 用户在桌面端打开首页
- **THEN** 右侧显示同心圆：核心"企业" + 内圈3节点（业务·财务·债务）+ 外圈3节点（资产重组·债务重组·业务重组）
- **AND** 节点均匀分布在各轨道上（120°间隔）
- **AND** 有连接线从核心指向内外节点
- **AND** 内外轨道环有不同速度的旋转

### Requirement: 悬停交互
- **WHEN** 用户悬停在任一节点上
- **THEN** 节点放大至 1.1x
- **AND** 显示简要说明 tooltip（如悬停"债务"显示"评估企业负债结构与偿债能力"）

### Requirement: 点击交互
- **WHEN** 用户点击任一节点
- **THEN** 在图表下方展开详细信息面板
- **AND** 面板包含该要素的详细说明文字

### Requirement: 加载性能
- **WHEN** 页面加载
- **THEN** 图表动画在 0.6s 内渐入完成
- **AND** 不影响页面首屏内容加载（LCP 不受影响）

## REMOVED Requirements
- 移除当前星系图结构（`.cosmic-core`、`.cosmic-ring-wrap`、`.cosmic-ring-line`、`.cosmic-inner-node`、`.cosmic-outer-node`、`.cosmic-dot`）
- 移除天体系统视觉隐喻

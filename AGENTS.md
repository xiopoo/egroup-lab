# HJ 项目协作规则

本仓库由用户、Trae 和 Codex 共同维护。任何 AI 或开发者开始任务前，必须先阅读根目录 `HJ_战略与开发总纲.md`，再阅读当前任务对应模块的 README、相关 `.trae/skills/` 与必要的 `.trae/specs/`。

## 项目事实

- 官网唯一正式源代码：`website/public/`
- 公众号内容资产：`content-system/content/`
- 公众号成品：`content-system/output/`
- Trae Skill 与规格唯一真源：根目录 `.trae/`
- 正式生产部署入口：根目录 `vercel.json`

不要把根目录残留 `public/` 当作正式官网，也不要在子目录重新创建另一套 `.trae` 真源。

## 工作方式

1. 先检查 Git 状态，保留用户和另一 IDE 已有的改动。
2. 先验证当前功能与真实文件，再制定修改方案。
3. 优先做小而可验证的修改，不无故重写完整网站或内容体系。
4. 业务规则、诊断评分、数据用途、价格和对外承诺不得由 AI 擅自改变。
5. 用户零编程基础：交付说明必须使用非技术语言，明确“完成了什么、如何验证、用户下一步做什么”。
6. 修改目录结构、删除文件、迁移数据或清理 Git 前，必须创建可恢复快照并取得用户确认。

## 验证要求

- 官网修改至少检查关键页面、链接、图片及手机/桌面基本显示；
- 内容修改遵循相关 Skill 的审核、法律检查、微信排版和保存流程；
- 涉及诊断或数据时检查授权、隐私、权限、日志、备份和删除路径；
- 涉及 AI 输出时保留来源、版本和人工复核入口；
- 生产部署、公众号发布、收费及真实客户数据操作必须由用户明确确认。

## 本地历史数据

当前数据目录为 `~/.hj/`。读取时若对应目录不存在，可只读回退到旧目录 `~/.regroup-lab/`；新增内容只写入 `~/.hj/`。未经用户确认，不得自动合并、覆盖或删除旧目录。

## 每次交付

最终说明至少包含：

- 本次完成；
- 用户现在能做什么；
- 验证结果；
- 尚存风险或未完成项；
- 建议下一步。
